Pedzpracovn dat

Provdn operace:

  1. Oeten chybjcch hodnot pomoc SimpleImputer (median pro numerick, most_frequent pro kategorick).
  2. Odvozen novch sloupc:
    • amount_per_month = credit_amount / duration
    • is_senior = 1 pokud age >= 60, jinak 0

Dle pouvme OneHotEncoder pro kategorick promnn a StandardScaler pro numerick. Ve je zabaleno do sklearn Pipeline/ColumnTransformer, aby nedolo k niku informac mezi trninkem a testem.

# Definice pedzpracovn: imputace + klovn (numerick), imputace + one-hot (kategorick)
numeric_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", StandardScaler())
])

categorical_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("onehot", OneHotEncoder(handle_unknown="ignore"))
])

preprocessor = ColumnTransformer(
    transformers=[
        ("num", numeric_transformer, num_cols),
        ("cat", categorical_transformer, cat_cols)
    ],
    remainder="drop"
)

# Rozdlen na trnink/test
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=SEED, stratify=y
)

def evaluate_model(name, model, X_test, y_test):
    y_pred = model.predict(X_test)
    acc = accuracy_score(y_test, y_pred)
    auc = None
    if hasattr(model, "predict_proba"):
        y_proba = model.predict_proba(X_test)[:, 1]
        auc = roc_auc_score(y_test, y_proba)
    print(f"{name} - Test Accuracy: {acc:.4f}" + (f", Test ROC AUC: {auc:.4f}" if auc is not None else ""))
    print("Confusion matrix:\n", confusion_matrix(y_test, y_pred))
    print("Classification report:\n", classification_report(y_test, y_pred))

def plot_confusion_matrices(models, X_test, y_test, normalize=None):
    n = len(models)
    fig, axes = plt.subplots(1, n, figsize=(5 * n, 4))
    if n == 1:
        axes = [axes]
    for ax, (name, est) in zip(axes, models):
        ConfusionMatrixDisplay.from_estimator(
            est, X_test, y_test,
            normalize=normalize,  # None | 'true' | 'pred' | 'all'
            cmap="Blues", ax=ax, colorbar=False
        )
        ax.set_title(name)
    fig.suptitle(f"Confusion matrices (normalize={normalize})")
    plt.tight_layout()
    plt.show()

def summarize_models(models, X_test, y_test):
    rows = []
    for name, est in models:
        y_pred = est.predict(X_test)
        acc = accuracy_score(y_test, y_pred)
        prec, rec, f1, _ = precision_recall_fscore_support(
            y_test, y_pred, average="weighted", zero_division=0
        )
        auc = np.nan
        if hasattr(est, "predict_proba"):
            try:
                auc = roc_auc_score(y_test, est.predict_proba(X_test)[:, 1])
            except Exception:
                auc = np.nan
        rows.append({
            "model": name,
            "accuracy": acc,
            "precision_w": prec,
            "recall_w": rec,
            "f1_w": f1,
            "roc_auc": auc,
        })
    return pd.DataFrame(rows)

summary_df = summarize_models(models_to_plot, X_test, y_test)

Vsledky a zvry

Analza ukzala, e pro predikci vrovho rizika na datech German Credit je nejspnjm modelem Random Forest, kter na testovac sad doshl ROC AUC piblin 0.78 a ven F1-skre 0.74. Tm vrazn pekonal jak zkladn baseline model (DummyClassifier), tak i optimalizovanou LogisticRegression.

Klov zjitn:

  • Vkon model: Random Forest prokzal nejlep schopnost generalizace na nevidnch datech. Logistick regrese byla slab, co naznauje, e vztahy v datech nejsou ist linern.
  • Dleitost pznak: Z analzy feature_importances_ modelu Random Forest vyplynulo, e nejvt predikn slu maj pznaky jako checking_status, duration, credit_amount a purpose. To dv smysl stav bnho tu, dlka a ve vru jsou logicky klov faktory pro banku.
  • Pedzpracovn: Pouit SMOTE pro vyrovnn td v trnovacch datech a klovn numerickch pznak bylo klov pro stabiln vkon model.
  • Feature Engineering: Vytvoen pznak amount_per_month a is_senior mlo mrn, ale pozitivn vliv.

Doporuen a dal kroky: Model Random Forest je solidnm zkladem pro praktick nasazen. Pro dal zlepen by bylo mon experimentovat s pokroilejmi modely (nap. Gradient Boosting - XGBoost, LightGBM) nebo zkusit sofistikovanj techniky feature engineeringu. Dle by bylo pnosn analyzovat chybovost modelu na jakch klientech model nejastji chybuje?

Cel proces byl navren s drazem na replikovatelnost (pevn random_state, Pipeline) a zamezen niku dat (train_test_split ped jakmkoliv fittovnm).
